Pesquisa sobre JavaScript

Quem criou o JavaScript e em que ano?

JavaScript foi criado por Brendan Eich em 1995 enquanto trabalhava na Netscape. Ele desenvolveu a linguagem em apenas 10 dias para ser usada no navegador Netscape Navigator 2.0​. (W3Schools.com).

Qual foi a motivação por trás da criação do JavaScript?

A motivação era criar uma linguagem simples e acessível para programadores iniciantes e designers, permitindo adicionar interatividade às páginas da web. JavaScript foi pensado como uma linguagem "cola" que pudesse ser usada para tarefas leves e dinâmicas no cliente, enquanto linguagens como Java lidavam com tarefas mais pesadas Cybercultural.

Em qual navegador o JavaScript foi implementado pela primeira vez?

O JavaScript foi implementado pela primeira vez no Netscape Navigator 2.0, em 1995​ (W3Schools.com).

Linha do tempo de eventos importantes na evolução do JavaScript

  • 1995: Criação do JavaScript por Brendan Eich.
  • 1996: Lançamento do Netscape Navigator 2.0 com suporte ao JavaScript 1.0.
  • 1997: O JavaScript se torna um padrão ECMAScript (ECMA-262).
  • 1999: Lançamento do ECMAScript 3 (ES3), uma versão crucial que definiu muitas das funcionalidades básicas usadas até hoje.
  • 2009: Lançamento do ECMAScript 5 (ES5), com melhorias em manipulação de objetos e arrays.
  • 2015: Lançamento do ECMAScript 6 (ES6), também conhecido como ES2015, que trouxe inovações como arrow functions, classes, e promises.
  • 2016-2023: Atualizações anuais, com novas versões trazendo funcionalidades como async/await (ES2017) e outras melhorias​ (W3Schools.com).

Frameworks e bibliotecas populares de JavaScript

React

O que faz: Biblioteca JavaScript usada para construir interfaces de usuário (UIs), especialmente em aplicações de página única. É baseada em componentes reutilizáveis.

Uso principal: Aplicações web interativas e dinâmicas. React é amplamente usado em desenvolvimento de front-end.

Angular

O que faz: Framework completo que fornece soluções para a criação de aplicações web robustas e escaláveis. Inclui tudo, desde templates até gerenciamento de rotas e injeção de dependências.

Uso principal: Aplicações corporativas de larga escala e sistemas que precisam de uma solução completa de front-end.

Vue.js

O que faz: Framework progressivo que pode ser adotado de forma incremental. Simples de usar, mas capaz de criar interfaces dinâmicas com facilidade.

Uso principal: Aplicações de front-end.
